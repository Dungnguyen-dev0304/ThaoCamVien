Tổng quan

Bối cảnh nghiệp vụ (Web + App)

  • Web Admin cần quản trị tập trung POI/tour/audio/bản dịch để đảm bảo dữ liệu chuẩn, đồng nhất đa ngôn ngữ.
  • App tham quan cần trải nghiệm ổn định theo trạng thái Loading/Success/Empty/Error, hỗ trợ bản đồ, audio, QR, offline.
  • Vận hành cần theo dõi thống kê hành vi người dùng trên App và hiệu quả nội dung trên Web để tối ưu liên tục.

Giải pháp tổng thể

  • WebThaoCamVien: xác thực quản trị, CRUD POI/tour, quản lý audio và translation, dashboard analytics.
  • AppThaoCamVien: Home/Map/StoryAudio/QR/About + onboarding, geofencing, narration, tải gói offline.
  • Kênh dữ liệu thống nhất qua API để App đọc nội dung đã được Web kiểm duyệt và xuất bản.
  • Hệ quy chiếu tài liệu đồng bộ bằng sequence diagrams + use case SVG cho cả Web và App.
Nguồn tham chiếu yêu cầu: sequence/use case của cả hai ứng dụng tại WebThaoCamVien/docs và AppThaoCamVien/Docs (xem mục Sequence diagrams và Use case).

Mục tiêu & KPI

Mục tiêu sản phẩm liên thông Web + App

  • Web: rút ngắn thời gian quản trị nội dung (POI/tour/audio/translation), giảm lỗi nhập liệu và lỗi file.
  • App: tăng mức độ hoàn tất hành trình tham quan (xem map, nghe thuyết minh, tra cứu QR) với trải nghiệm mượt.
  • Vận hành: có dashboard đo được hiệu quả nội dung và hành vi người dùng để ra quyết định nhanh.

KPI đề xuất (tách Web/App)

KPI Định nghĩa Mục tiêu
Web - Tỷ lệ POI đủ nội dung xuất bản % POI có ảnh + tọa độ hợp lệ + audio tối thiểu 1 ngôn ngữ + bản dịch mặc định ≥ 95%
Web - Thời gian xử lý nội dung Thời gian trung bình từ tạo mới POI đến trạng thái sẵn sàng cho App ≤ 3 phút
Web - Tỷ lệ lỗi thao tác quản trị % thao tác thất bại do validate/model/file/upload ≤ 3%
App - Tỷ lệ mở thành công màn hình chính % phiên mở Home/Map không rơi vào Error state ≥ 98%
App - Tỷ lệ tra cứu POI thành công % lượt QR/Numpad tìm thấy POI hợp lệ và mở được StoryAudio ≥ 90%
Web + App - Mức độ sử dụng analytics Số lượt xem dashboard + số bản ghi playback log hợp lệ theo tuần Theo mùa cao điểm

Phạm vi & ngoài phạm vi

Trong phạm vi In scope

  • Đăng nhập/đăng xuất bằng Cookie Authentication.
  • Tour: xem danh sách, thêm, sửa, bật/tắt active, xóa.
  • POI: danh sách, thêm/sửa/xóa; upload ảnh thumbnail; validate tọa độ (ranh giới + khoảng cách tối thiểu).
  • Audio theo POI: danh sách, quản lý theo POI; upload file audio; record trực tiếp trên web (lưu .webm); xóa audio.
  • Bản dịch POI: thêm/cập nhật theo ngôn ngữ; trang quản lý tổng hợp.
  • Admin Index/Analytics: thống kê + lịch sử truy cập, lọc theo POI và số ngày, phân trang.

Ngoài phạm vi Out of scope

  • Tối ưu hóa tìm kiếm nâng cao (full-text), recommendation tour tự động.
  • Streaming audio nâng cao, transcoding đa định dạng trên server.
  • Ứng dụng mobile/clients cho khách tham quan (nếu có sẽ là PRD riêng).
  • Phân quyền đa vai trò phức tạp (RBAC nhiều cấp) — hiện coi là “Admin” truy cập.

Pain points

Mục tiêu

Khách tham quan (App User)

Mục tiêu
Dễ tìm điểm tham quan, nghe thuyết minh, tra cứu nhanh bằng QR hoặc mã.
Pain points
Mạng không ổn định, khó định vị, nội dung không đồng nhất theo ngôn ngữ.

Nhóm vận hành kỹ thuật

Mục tiêu
Giám sát lỗi App/Web, đảm bảo dữ liệu đồng bộ API - DB - file.
Pain points
Khó truy vết lỗi liên thông giữa backend, dashboard, và app client.

Giá trị mang lại

  • Giá trị cho Web: tăng năng suất quản trị nội dung, giảm thao tác lỗi và cải thiện khả năng kiểm soát vòng đời dữ liệu.
  • Giá trị cho App: trải nghiệm tham quan liền mạch qua map, geofencing, narration, QR/Numpad và cơ chế offline.
  • Giá trị vận hành: dữ liệu analytics đồng nhất giúp đo chất lượng nội dung và hành vi người dùng theo thời gian.
  • Giá trị mở rộng: kiến trúc API-first giúp mở rộng thêm ngôn ngữ, loại nội dung và kênh client trong tương lai.

Yêu cầu chức năng (Web + App)

Ghi chú: Mục này chỉ mô tả yêu cầu chức năng Web + App. Sequence diagrams được tách sang mục riêng bên dưới.

FR-01 — Đăng nhập (Cookie Authentication)

  • GET /Account/Login: nếu đã đăng nhập thì redirect /Admin/Index, nếu chưa thì hiển thị form.
  • POST /Account/Login: nhận email/password + anti-forgery; xác thực với DB; tạo cookie (persistent) thời hạn ~7 ngày; lỗi thì trả về view với thông báo.

FR-02 — Đăng xuất

  • Cho phép admin đăng xuất, xoá session/cookie, về trang login (hoặc trang công khai nếu có).

FR-03 — Tour management

Tính năng Yêu cầu Ghi chú
Danh sách tour Xem danh sách, trạng thái active, thao tác sửa/xóa/bật-tắt. Tham chiếu: sequence diagrams “03…06”.
Thêm tour Form tạo mới; validate dữ liệu bắt buộc; lưu DB. Định nghĩa trường cụ thể theo model hiện có.
Sửa tour Load theo id; chỉnh sửa; validate; lưu DB.
Bật/tắt active Toggle hiển thị tour; phản hồi rõ ràng trạng thái sau khi cập nhật.
Xóa tour Xóa khỏi DB (hoặc soft delete nếu đang áp dụng). Tham chiếu “06-tour-toggle-delete”.

FR-04 — POI management (CRUD + ảnh + validate tọa độ)

  • Danh sách POI: sắp xếp mới nhất trước; hiển thị POI mới tạo nổi bật (tham chiếu ViewBag.NewPoi trong diagram).
  • Thêm POI: bắt buộc ảnh thumbnail; validate ModelState; validate tọa độ trong ranh giới; kiểm tra khoảng cách tối thiểu với POI khác (diagram đề cập < 5m).
  • Sửa POI: validate tương tự; nếu đổi ảnh thì xóa ảnh cũ rồi lưu ảnh mới với tên file unique.
  • Xóa POI: xác nhận; xóa ảnh thumbnail (nếu có) trên filesystem; xóa record DB.
Tiêu chí chấp nhận (AC) gợi ý: (1) Không thể lưu POI ngoài ranh giới; (2) Không thể lưu POI quá gần POI hiện hữu theo ngưỡng cấu hình; (3) Không để “mồ côi” file ảnh/audio sau xóa record.

FR-05 — Audio theo POI (upload / record / delete)

  • Trang danh sách audio: hiển thị POI + map audio theo ngôn ngữ + danh sách ngôn ngữ hỗ trợ.
  • Trang quản lý theo POI: xem các bản audio hiện có theo ngôn ngữ.
  • Upload audio: validate file (không rỗng, đúng định dạng); nếu đã có audio cùng (POI, LanguageCode) thì xóa file cũ và update record; nếu chưa có thì insert mới.
  • Record trên web: ghi âm và lưu blob (.webm); hành vi cập nhật/tạo mới tương tự upload; trả JSON {success:true/false}.
  • Xóa audio: xóa file nếu tồn tại + xóa record DB.

FR-06 — Translation management (thêm/cập nhật bản dịch POI)

  • Trang quản lý tổng hợp: xem POI và các bản dịch theo ngôn ngữ.
  • Thêm bản dịch: tạo mới theo (POI, LanguageCode) nếu chưa có.
  • Cập nhật bản dịch: chỉnh sửa nội dung bản dịch hiện có.

FR-07 — Admin Index / Analytics (thống kê + lịch sử)

  • GET /Admin/Index?poiId&days&page hỗ trợ lọc theo POI và số ngày gần đây.
  • Thống kê gợi ý theo diagram: totalVisits, totalUsers, todayVisits, average listen duration (nếu có), chuỗi 7 ngày gần nhất, Top 5 POI, danh sách lịch sử gần đây (phân trang).
  • Hiển thị danh sách POI để lọc nhanh.

Phần App — Yêu cầu chức năng

Phần App được tổng hợp từ đặc tả UX theo hướng API-first tại AppThaoCamVien/Docs/ApiFirstUxSpec.md. Các sequence diagrams cho App hiện chưa có (sẽ bổ sung sau).

AFR-01 — State machine chuẩn cho mọi màn hình

  • Loading: hiển thị skeleton, không để màn hình trống.
  • Success: hiển thị dữ liệu bind từ API.
  • Empty: API trả 200 nhưng payload rỗng.
  • Error: lỗi mạng/timeout/server, có CTA “Retry”.

AFR-02 — Home

  • Dữ liệu: HomeFeed, Weather.
  • Empty: không có card ở cả HappenNow và Recommended.
  • Error CTA: retry + mở map như phương án fallback.

AFR-03 — Map

  • Dữ liệu: PoiClusters, NearbyAnimals.
  • Empty: không có POI trong viewport.
  • Error CTA: retry + vẫn giữ cached map.

AFR-04 — StoryAudio

  • Dữ liệu: Voices, LyricsTimeline.
  • Empty: không có script/timeline.
  • Error CTA: retry + fallback (studio audio/TTS) nếu cấu hình cho phép.

AFR-05 — QR

  • Dữ liệu: QrLookup.
  • Empty: code không tồn tại (404/null).
  • Error CTA: retry scan + cho nhập tay mã.

AFR-06 — About

  • Dữ liệu: AboutSections.
  • Empty: danh sách section rỗng.
  • Error CTA: retry + hiển thị thông tin liên hệ fallback.

AFR-07 — Skeleton component catalog (tối thiểu)

  • SkeletonBlock, StateContainer
  • Compositions gợi ý: HomeHeaderSkeleton, HomeHorizontalCardSkeleton, MapNearbyListSkeleton, StoryAudioSkeleton, AboutAccordionSkeleton

Yêu cầu phi chức năng

Hiệu năng (Web + App)

  • Web: trang danh sách POI/tour/audio tải ≤ 2s với dữ liệu quy mô vừa, có phân trang và lọc hiệu quả.
  • Web: upload/record audio có phản hồi tiến trình rõ ràng; thao tác hoàn tất ≤ 10s tùy kích thước file.
  • App: thời gian hiển thị nội dung đầu tiên (Home/Map) mục tiêu ≤ 2.5s trong điều kiện mạng ổn định.
  • App: chuyển trạng thái Loading → Success/Empty/Error nhất quán, không treo UI.

Độ tin cậy & phục hồi

  • Web: không mất đồng bộ DB ↔ filesystem; nếu lưu DB thất bại phải dọn file mới, nếu lưu file thất bại không ghi DB.
  • App: hỗ trợ retry khi lỗi mạng; fallback dữ liệu cục bộ cho các luồng cho phép (map/offline/about).
  • Web + App: ghi log cho thao tác quan trọng (đăng nhập, CRUD, phát audio, lỗi hệ thống) để truy vết sự cố.
  • Duy trì version history tối thiểu cho POI/Tour/Translation để tra cứu và rollback khi cần.

Khả năng mở rộng

  • Ngôn ngữ hỗ trợ được cấu hình; audio/translation theo LanguageCode cho cả Web và App.
  • Cho phép thay đổi ngưỡng geofencing/radius và khoảng cách tối thiểu POI theo cấu hình.
  • Kiến trúc API-first cho phép mở rộng thêm module App và dashboard Web mà không phá vỡ luồng cũ.

Khả dụng & truy cập

  • Web: thông báo thành công/thất bại rõ ràng (TempData/ValidationSummary), không nuốt lỗi.
  • App: luôn có màn hình Empty/Error kèm CTA rõ ràng (Retry, mở map, nhập mã thủ công...).
  • Form Web và màn hình App ưu tiên khả năng đọc tốt, cỡ chữ phù hợp, điều hướng nhất quán.

Backup & khôi phục

  • Sao lưu định kỳ dữ liệu Web (POI, tour, audio metadata, bản dịch, logs) theo lịch và có quy trình restore rõ ràng.
  • App hỗ trợ phục hồi phiên bản dữ liệu offline theo gói tải, tránh hỏng cache khi gián đoạn mạng.

Dựa trên use case và sequence của Web + App, hệ thống tối thiểu có các thực thể sau (tên bảng/thuộc tính chi tiết theo code/DB thực tế):

Bảo mật & phân quyền

  • Authentication (Web): Cookie Authentication + anti-forgery cho các POST form quản trị.
  • Authorization (Web): toàn bộ route quản trị yêu cầu đăng nhập; role vận hành hiện tại chuẩn hóa về Admin.
  • Client security (App): kiểm soát quyền camera/GPS/thông báo theo đúng luồng onboarding, từ chối quyền vẫn phải hoạt động an toàn.
  • API security: xác thực request từ App, kiểm soát CORS/rate limit, hạn chế lộ thông tin nội bộ qua lỗi chi tiết.
  • File upload: kiểm tra định dạng + kích thước, đặt tên file unique, chặn path traversal và file độc hại.
  • Password: bắt buộc hash + salt, không lưu plain-text.
Backlog bảo mật: harden upload, rate-limit login/API, lockout, audit log đầy đủ, ký request App bằng token ngắn hạn.

UX/UI & quy ước nội dung

  • Web: thông báo thao tác thành công/thất bại nhất quán; form POI/audio/translation trả lỗi chi tiết theo field.
  • Web: bảng dữ liệu ưu tiên lọc, sắp xếp, phân trang rõ ràng; dashboard hiển thị KPI chính trước.
  • App: tất cả màn hình tuân thủ state machine Loading/Success/Empty/Error với CTA rõ ràng.
  • App: map và story audio ưu tiên thao tác một tay, icon rõ nghĩa, trạng thái phát/tạm dừng dễ nhận biết.
  • Nội dung đa ngôn ngữ cần thống nhất thuật ngữ giữa CMS và App để tránh lệch bản dịch.

Rủi ro & giả định

Giả định

  • Web Admin là hệ thống nội bộ, không mở public cho khách tham quan.
  • App sử dụng chung nguồn dữ liệu đã được xuất bản từ Web, không chỉnh sửa trực tiếp dữ liệu lõi.
  • Ngưỡng geofencing và khoảng cách tối thiểu POI là tham số nghiệp vụ có thể cấu hình.
  • Danh sách ngôn ngữ hỗ trợ được định nghĩa sẵn và đồng nhất giữa Web và App.

Rủi ro

  • High Lỗi bảo mật xác thực/phân quyền có thể làm lộ dữ liệu quản trị hoặc API nội bộ.
  • Medium Quản lý file trên filesystem dễ sinh file mồ côi nếu transaction không chặt.
  • Medium Mạng di động không ổn định khiến App dễ rơi vào Error/timeout nếu thiếu fallback.
  • Medium Trôi dữ liệu giữa Web xuất bản và App cache offline nếu thiếu cơ chế version bundle.
  • Medium Analytics query nặng khi dữ liệu lớn cần index, phân trang và archive định kỳ.

Phụ lục

Định nghĩa thuật ngữ

  • POI (Point of Interest): điểm tham quan trong khuôn viên.
  • LanguageCode: mã ngôn ngữ (ví dụ: vi, en, fr…).
  • VisitHistory: lịch sử truy cập/nghe tại POI (có thể kèm thời lượng nghe).

Backlog gợi ý (nâng cấp)

  • Chuẩn hóa bảo mật: hash password, lockout, rate limit.
  • Thêm soft delete + audit log cho POI/Tour/Audio/Translation.
  • Chuyển file storage sang object storage (S3/MinIO/Azure Blob) nếu cần scale.
  • Thêm kiểm tra format audio + tự chuyển đổi (transcode) nếu triển khai server-side.
